# o24キーバインド変更ルール

このドキュメントは、Karabiner-Elementsの`o24`プロファイルを使うアプリケーションの
キーバインドを変更するときの共通ルールです。キーバインドを追加・変更・レビュー
する前に、必ず全文を確認してください。

## 最重要ルール

アプリケーションの設定には、Karabiner変換後にアプリケーションが受け取る
o24入力文字をそのまま記述します。QWERTYの物理位置へ逆変換しません。

たとえば、上移動をo24入力の`n`へ割り当てる場合は次のようになります。

```text
QWERTYの物理位置 k
  -> Karabinerが n へ変換
  -> アプリケーションが n を受信
  -> 設定値は n
```

次の設定は誤りです。

```text
QWERTYの物理位置 n
  -> Karabinerが g へ変換
  -> 設定値を g にする
```

この誤った変換では、o24入力の`n`ではなくQWERTYの物理`n`の位置で操作が
発火してしまいます。

物理位置を基準にする設定は、ユーザーが明示的に「QWERTYの物理位置」または
「Spaceレイヤーの物理キー」を指定した場合だけ使用します。

## 設定の読込先

OSや一般的な既定値から設定パスを推測せず、現在インストールされている
アプリケーション自身が返すパスを確認します。設定ファイルを変更しても、
実際の読込先にリンクされていなければキーバインドは反映されません。

```sh
lazygit --print-config-dir
codex-history config path
```

アプリにパス表示コマンドがない場合は、公式の診断コマンド、起動ログ、開いている
ファイルの確認など、実行中バージョンの情報で特定します。READMEと`setup.sh`の
リンク先も、確認した実パスへ一致させます。

## 用語

| 用語 | 意味 | 例 |
| --- | --- | --- |
| QWERTYの物理位置 | キーキャップ上または標準配列上の位置 | 物理`k` |
| o24入力文字 | Karabiner変換後にアプリが受け取る文字 | `n` |
| アプリ設定値 | keybindingへ記述する値 | `n` |
| Spaceレイヤー出力 | Karabinerのcomplex modificationが直接送るキー | `Left` |

READMEや設定コメントでは、原則として「o24入力」と表記します。「物理キー」と
表記する場合は、物理位置を使う明確な理由も記載します。

## 変更前の必須確認

1. このドキュメントを全文読む。
2. ユーザーの希望が「o24入力文字」か、明示的な「物理位置」かを確認する。
3. 対象アプリ自身のコマンドで、現在の設定読込先を確認する。
4. 対象アプリが受け取るキー表現と、グローバル・画面固有の競合を確認する。
5. `Ctrl+c`、矢印、Enter、Escなどの復旧手段を可能な範囲で残す。
6. 設定ファイル、`setup.sh`のリンク先、対応するREADMEを同じ変更で更新する。

意図が曖昧なときは、たとえば「上移動はo24で`n`と入力されるキーですか、
それともQWERTYの物理`n`の位置ですか」のように、具体例で確認します。

## 変更後の必須検証

1. 対象アプリの設定検証または構文検証を実行する。
2. 同時に有効なスコープのキー競合を確認する。
3. アプリが返す設定ディレクトリと設定リンクの配置先が一致し、リンクが意図した
   リポジトリ内ファイルを指すことを確認する。
4. 起動中のアプリを再起動し、o24入力文字と矢印などの復旧キーを実機で確認する。
5. `git diff --check`と、リポジトリの`AGENTS.md`にある検証を実行する。

設定変更前から起動しているアプリは古いキーマップを保持している場合があります。
再読込キー自体を変更したときは、再読込に頼らず一度終了して起動し直します。

## 現在の管理方針

- codex-history: o24入力の`n/t/k/s`を移動へ使用する。
- Lazygit: o24入力の`n/t/k/s`を移動へ使用し、競合操作はRight Optionへ移す。
- Nixvim: すでにo24入力の`k/t/n/s`を直接マッピングしている。
- WezTerm: 入力文字または名前付きキーを直接設定し、QWERTY物理位置への逆変換は
  行わない。

Karabiner本体の配列を変更した場合も、アプリ設定を機械的に逆変換しません。
各アプリで使いたいo24入力文字を基準に、設定とREADMEを見直します。
